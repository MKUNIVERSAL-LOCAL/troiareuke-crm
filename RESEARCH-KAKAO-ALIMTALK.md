# 카카오톡 알림톡 중계사 조사 리포트

- **작성일**: 2026-04-17
- **대상 프로덕트**: 트로이아르케 CRM (에스테틱 CRM, Phase 0 MVP)
- **목적**: 고객 자동 재방문 리마인더 알림톡 발송
- **발송 규모 가정**: 월 100 ~ 5,000건 (초기), 단일 카카오톡 채널 1개
- **요건**: 알림톡 API, 카카오 비즈채널 연동, Node.js 친화
- **주의**: 이 리포트는 공개 웹에서 확인 가능한 정보만 포함. 확인 못한 항목은 "확인 필요"로 명시. 모든 단가는 별도 명시 없으면 **VAT 별도**.

---

## 1. 카카오 알림톡 기본 배경 (공통)

- 알림톡은 **정보성** 메시지 (예약 리마인더, 재방문 안내 등). 광고성은 친구톡/브랜드 메시지.
- 발송하려면 반드시: (1) **카카오톡 채널 개설** → (2) **비즈니스 채널 인증** → (3) **중계사 연동 및 발신프로필(PFID/Sender Key) 발급** → (4) **템플릿 카카오 심사 승인** 이후 발송 가능.
- 채널 비즈니스 인증: 영업일 **1~3일** 소요 (사업자등록증·대표자 신분증 필수).
- 템플릿 심사: 카카오 본사 검수, 영업일 **1~3일** (중계사에 따라 2~5일 언급). 반려 시 재작성 후 재심사.
- 전체 가입~첫 발송까지 현실적으로 **2~3주** 버퍼 권장. 빠르면 1주 내 가능.

출처: SOLAPI 알림톡 가이드, 알리고 도움말, kakao business gitbook, 비즈고 블로그 "알림톡 1주일 세팅" 등.

## 2. 서비스별 상세

### 2.1 SOLAPI (舊 쿨에스엠에스, 솔라피)

- **한 줄 요약**: 국내 중계사 중 Node.js/개발자 경험(DX) 1등급. 공식 npm SDK, 후불 결제, 모던 API 문서.
- **건당 가격 (알림톡 / 친구톡 / SMS / LMS)**:
  - 웹/콘솔 기본 단가: 알림톡 **13원**, 월 발송량 구간별 최대 57.69% 할인 (10,000건 이상 10원, 이후 단계적 할인).
  - **직접 API 연동 단가**: 알림톡 **8원**, 친구톡 14원, 친구톡 이미지 22원, SMS 13원, LMS 29원, MMS 60원.
- **월 기본료 / 최소 사용료**: 월 기본료 없음. API 사용료 없음. 최소 사용료 없음 (선불 충전 또는 후불).
- **가입 난이도**: 이메일·휴대폰으로 가입 후 가입 즉시 300P 테스트 포인트 제공. 알림톡 사용 시 사업자등록증 + 카카오 비즈채널 필수.
- **API 문서/SDK**: Node.js 공식 SDK (`npm i solapi`, `github.com/solapi/solapi-nodejs`). TypeScript 지원. PHP, Python, Java, .NET, Ruby, Go SDK 제공. `examples/javascript/common/src/kakao/send/send_alimtalk.js` 공식 예제 존재. 문서 수준 중계사 중 최상.
- **가입→발송 소요**: 가입 당일 SMS 테스트 발송 가능. 알림톡은 비즈채널 인증 + 템플릿 심사 포함 **1~2주**.
- **템플릿 등록**: 솔라피 콘솔 또는 `createKakaoAlimtalkTemplate` API. 카카오 심사 1~3 영업일.
- **리포팅**: 웹 대시보드에서 발송 상태·성공률·리포트 조회. Webhook 지원.
- **평판**: 스타트업/개발자 커뮤니티에서 "반나절 만에 API 연동 끝냈다"는 평 다수. CoolSMS 인지도를 그대로 계승, 국내 중계사 중 DX 평이 가장 높음.
- **강점**: (1) Node.js SDK 품질, (2) 후불결제, (3) 월 기본료 없음, (4) 직접 연동 단가 8원으로 경쟁력 있음.
- **약점**: 웹 콘솔 기본 단가(13원)가 대량 발송이 없으면 소규모엔 덜 매력적. 다만 직접 연동 단가로 가면 경쟁력 있음.

출처: `solapi.com/pricing`, `developers.solapi.dev`, `github.com/solapi/solapi-nodejs`, `solapi.zendesk.com/hc/ko/articles/360053238434`.

### 2.2 알리고 (Aligo, 알리는사람들)

- **한 줄 요약**: 국내 최저가 포지셔닝. 단가는 낮지만 DX는 올드 PHP 스타일. 개인·소규모 쇼핑몰 다수 사용.
- **건당 가격**:
  - 알림톡 **6.5원** (공식 랜딩 기재, VAT 별도).
  - 친구톡 텍스트형 12.5원, 친구톡 이미지 19.5원.
  - SMS 8.4원, LMS 25원, MMS 60원.
- **월 기본료 / 최소 사용료**: 월 기본료 **없음**. 선불 충전 방식 (가상계좌 입금). 최소 충전액은 **확인 필요** (공지 상 일반적으로 1만원 내외로 알려짐, 공식 확정 수치 미확인).
- **가입 난이도**: 가입 자체는 무료. 알림톡 사용 위해 사업자 인증(사업자등록증 또는 고유번호증) 필수 + 카카오 비즈채널.
- **API 문서/SDK**: REST API. 공식 예제는 PHP, JSP, .NET, Node.js, Python 다운로드 제공. **공식 npm 패키지 없음** (커뮤니티 패키지 `aligo-kakao-api` 등 GitHub에 존재). 문서 스타일이 다소 올드 (HTML 페이지 기반).
- **가입→발송 소요**: 카카오 심사 2~4 영업일, 템플릿 검수 1~3일. 중계사 자체의 계정 지연은 거의 없음.
- **템플릿 등록**: [카카오톡 알림톡] → [템플릿 관리] 메뉴에서 등록, 카카오 검수.
- **리포팅**: 웹 관리자 화면 기본 제공. 발송 실패 시 자동 환불(공식 기재).
- **평판**: 가격이 싸서 소규모 쇼핑몰·자영업자·스마트스토어 연계 자동화 업체들에서 널리 사용. 반면 Node.js 개발자 커뮤니티에서 "공식 SDK가 없고 문서가 오래됨" 언급 다수.
- **강점**: (1) 알림톡 6.5원으로 이 리포트 조사 대상 중 **최저가 권** (직접 연동 기준), (2) 발송 실패 자동 환불, (3) 가입 장벽 낮음.
- **약점**: (1) 공식 Node.js SDK 부재, (2) 문서 스타일 구식, (3) 후불 결제 비공식 (선불 중심).

출처: `smartsms.aligo.in/aligoxkakao.html`, `smartsms.aligo.in/alimapi.html`, `smartsms.aligo.in/shop/popup_biz_auth.html`.

### 2.3 비즈엠 (BizM, 스윗트래커 운영)

- **한 줄 요약**: 카카오 공식 딜러사 중 대형. 이커머스/대형 고객(스마트스토어 등) 중심. 개인·SMB는 딜러사 재판매 경로.
- **건당 가격**: 공식 사이트에 **공개 요금표 없음** ("상담 문의" 형태). 검색상 전형적인 공식 딜러사 가격대는 알림톡 8~10원, 기본료 1만원대 (같은 스윗트래커 계열 대리점 "비즈톡"의 경우: 기본 이용료 1만원 + 건당 8원 구조 확인됨). **정확 요금은 확인 필요**.
- **월 기본료 / 최소 사용료**: 대리점/플랜별 상이, 통상 월 기본료 있음. **확인 필요**.
- **가입 난이도**: 사업자등록증 + 비즈채널 필수. 딜러사 선택 후 가입 → 비즈니스 인증 → 채널 ID 등록의 다단계. 개인/1인샵은 보통 서브 딜러사를 통함.
- **API 문서/SDK**: REST API 문서(`alimtalk-api.bizmsg.kr/codeList.html`) 공개, 하지만 **공식 Node.js SDK 없음**. 문서는 결과코드 리스트 수준으로 제한적, 딜러사 개발자 가이드에 의존.
- **가입→발송 소요**: 딜러사 계약·비즈채널 인증까지 포함해 일반적으로 **1~3주**. 개인 개발자가 셀프 온보딩 하기엔 무거움.
- **템플릿 등록**: 딜러사 관리자 페이지 경유, 카카오 심사.
- **리포팅**: 웹 관리자 사이트 제공.
- **평판**: 대형 커머스·물류 고객 기반으로 안정적이나, **스타트업 1인 개발자가 처음 붙기엔 과하게 엔터프라이즈 성향**이라는 후기 다수.
- **강점**: (1) 발송 안정성·처리량(대형사 레퍼런스), (2) 공식 딜러사.
- **약점**: (1) 공개 요금 정보 부족, (2) Node.js SDK 없음, (3) 대리점 경유 필요해 가입 복잡, (4) MVP급 초기 사용에는 오버킬.

출처: `bizmsg.kr`, `alimtalk-api.bizmsg.kr/codeList.html`, `biztalk.co.kr` (계열 대리점 가격 참조).

### 2.4 NHN Cloud 알림톡 (KakaoTalk Bizmessage)

- **한 줄 요약**: NHN Cloud의 Notification 서비스 한 축. 클라우드 플랫폼 기반, IT 내재화 규모 조직에 적합.
- **건당 가격**: 공식 요금 페이지(`nhncloud.com/kr/pricing/calculator?c=notification&s=KakaoTalk+Bizmessage`)에 있으나 **본 조사에서 구체 건당 가격은 확인하지 못함** (접근한 검색 결과에서 수치 미확인). **확인 필요** (다만 NHN·네이버 같은 클라우드형은 통상 건당 9~11원대로 알려짐 — 정확 수치 아님).
- **월 기본료 / 최소 사용료**: NHN Cloud 표준 사용량 기반 과금. 별도 고정 기본료는 없지만 **확인 필요**.
- **가입 난이도**: NHN Cloud 계정 가입 → 프로젝트 생성 → Notification 서비스 활성화. 사업자등록증 + 비즈채널 필요.
- **API 문서/SDK**: REST API 문서 상세 (`docs.nhncloud.com/ko/Notification/KakaoTalk Bizmessage`). **공식 Node.js SDK 부재**, REST 직접 호출 방식. API 문서는 체계적.
- **가입→발송 소요**: 콘솔 세팅 자체는 1~2일, 카카오 심사 포함 1~2주.
- **템플릿 등록**: 콘솔에서 등록 → 카카오 심사 (콘솔 사용 가이드 공식 문서 제공).
- **리포팅**: 콘솔 통계·Webhook 제공. 로그 품질 준수한 편.
- **평판**: 대형 쇼핑몰/엔터프라이즈에서 선호. 스타트업 초기 팀에선 "콘솔 구조가 복잡, 클라우드 리소스 붙여야 함" 의견.
- **강점**: (1) 클라우드 네이티브, (2) 문서 체계, (3) 타 NHN Cloud 서비스와 통합 쉬움.
- **약점**: (1) 공식 Node.js SDK 없음, (2) 콘솔 러닝 커브, (3) 공개 단가 정보가 즉시 보이지 않아 의사결정에 마찰.

출처: `nhncloud.com/kr/service/notification/kakaotalk-bizmessage`, `docs.nhncloud.com/ko/Notification/KakaoTalk Bizmessage/ko/alimtalk-overview/`.

### 2.5 네이버 클라우드 SENS (Simple & Easy Notification Service)

- **한 줄 요약**: 네이버 클라우드 생태계. 이미 네이버 클라우드 쓰는 팀이면 자연스러운 선택.
- **건당 가격**: SENS 알림톡은 건당 과금, **구체 건당 수치는 이 조사에서 직접 확인 불가** (공식 요금 페이지 수치 매핑 실패). **확인 필요**. 커뮤니티상 SOLAPI 대비 비슷하거나 약간 비싼 편으로 언급되나 수치 근거 없음.
- **월 기본료 / 최소 사용료**: 네이버 클라우드 표준, 별도 고정 기본료는 없음으로 알려짐. **확인 필요**.
- **가입 난이도**: 네이버 클라우드 플랫폼 가입 → 결제수단 등록 → SENS 서비스 활성화. 카카오 비즈채널 필수.
- **API 문서/SDK**: REST API, 가이드 문서 공식 제공(`api.ncloud-docs.com/docs/ai-application-service-sens-alimtalkv2`). **공식 Node.js SDK 없음**. 커뮤니티 예제(velog 등) 다수. HMAC 서명 필요해 초기 구현 난이도 중.
- **가입→발송 소요**: 계정 생성 및 결제 등록 하루, 심사 포함 1~2주. **월 30,000건 이용 한도** 기본 적용 (요청 시 상향 가능).
- **템플릿 등록**: 콘솔에서 등록, 카카오 심사.
- **리포팅**: 콘솔 통계·Webhook.
- **평판**: 네이버 클라우드를 이미 쓰는 팀에게 편리. DX는 NHN Cloud와 비슷한 수준. 국내 레퍼런스 많음.
- **강점**: (1) 네이버 생태계 통합, (2) 문서 체계, (3) 인프라 안정성.
- **약점**: (1) 공식 Node.js SDK 없음, (2) HMAC 서명 등 초기 복잡도, (3) 단가 정보 투명성 낮음, (4) 월 30,000건 기본 한도.

출처: `ncloud.com/v2/product/applicationService/sens`, `api.ncloud-docs.com/docs/ai-application-service-sens-alimtalkv2`, `guide.ncloud-docs.com/docs/sens-atmessage`.

### 2.6 카카오 비즈메시지 Direct (카카오 공식 API)

- **한 줄 요약**: 카카오가 직접 제공하는 파트너사용 API. 사실상 **일반 스타트업/SMB가 직접 가입할 수 없음**.
- **건당 가격**: 카카오 본사 공식 단가 (알림톡 약 7~9원대)로 알려지나 본 조사에서 직접 확인 실패. **확인 필요**. 일반적으로 중계사들이 본사 단가에 마진을 붙여 재판매.
- **월 기본료 / 최소 사용료**: 카카오 파트너 계약 기준 적용. **확인 필요**.
- **가입 난이도**: 카카오와 **파트너사 계약 필요**. 개별 사업자(특히 초기 스타트업)가 직접 계약하기 사실상 불가. 카카오 공식 안내: "알림톡은 카카오와 파트너 계약이 되어있는 공식 딜러사를 통해 진행 가능".
- **API 문서/SDK**: `bizmsg-web.kakaoenterprise.com/api-docs/` (카카오엔터프라이즈). 파트너 전용 접근.
- **평판**: 본사 직접 연동은 월 수십만 건 이상 규모 사업자·대형 ISV에서 의미 있음.
- **강점**: 가장 안정적인 원천. 중간 마진 없음.
- **약점**: **개인·초기 스타트업이 접근 불가**. MVP 후보에서 **제외**.

출처: `us.business.kakao.com`, `cs.kakao.com/helps_html/1073194757`, `bizmsg-web.kakaoenterprise.com/api-docs/`.

## 4. 종합 순위 (트로이아르케 CRM MVP 기준)

**1위. SOLAPI** — Node.js SDK 공식 지원, 월 기본료 없음, 후불 결제, 직접 연동 8원으로 가격 경쟁력 충분. 개발 시간 절감 효과가 건당 단가 차이(1.5원)를 압도. 월 5,000건 기준 월 약 **40,000원** 수준 (8원 × 5,000).

**2위. 알리고** — 최저 건당 단가(6.5원). 월 5,000건 기준 월 약 **32,500원**. 다만 공식 Node.js SDK 부재로 커뮤니티 패키지 또는 자체 REST 구현 필요 → 초기 개발/유지보수 부담 약간 증가.

**3위. NHN Cloud 알림톡** — 이미 NHN Cloud 인프라를 쓰거나, 장기적으로 대량 발송·엔터프라이즈 고객까지 커버할 계획이면 유효. MVP 단계엔 오버킬.

## 5. MVP 최종 추천: **SOLAPI**

### 추천 이유 (왜 SOLAPI인가)

1. **Node.js 공식 SDK가 있는 유일한 중계사**. 트로이아르케 CRM은 Node.js 기반, SDK로 통합하면 REST 서명/재시도/템플릿 관리까지 캡슐화됨. 개발 시간 **하루 이내** 기대.
2. **월 기본료 0원, 최소 사용료 0원**. 초기 고객 수 적을 때 고정비 부담 없음. 후불 결제도 가능해 잔액 관리 부담 없음.
3. **직접 연동 단가 알림톡 8원** — 알리고의 6.5원보다 1.5원 비싸지만, 월 5,000건에서 차액은 **월 7,500원**. 이 차이로 공식 SDK의 이점(타입 정의, 공식 유지보수, 문서 품질)을 확보하는 편이 MVP 시점에 명백히 합리적.
4. **문서·예제·템플릿 관리 API까지 Node.js로 자동화 가능** — CRM에서 템플릿 프로비저닝을 코드로 관리하는 시나리오에 핵심.
5. **확장성**: 향후 SMS, LMS, 친구톡(프로모션)으로 채널 확장 시 동일 SDK·동일 계정으로 처리 가능.

### 가격 변곡점 모니터링

- 월 10,000건을 넘기기 시작하면 SOLAPI 구간별 할인(콘솔 기본 단가에서) 또는 알리고로의 교체 고려. 다만 직접 연동 8원 자체가 이미 낮은 수준.
- 월 50,000건 이상으로 크면 NHN Cloud/네이버 SENS 또는 비즈엠 공식 딜러와 별도 협상 검토.

### 위험 및 완충책

- **벤더 락인**: SDK를 CRM의 `src/integrations/kakao/` 어댑터 레이어로 격리. 내부 인터페이스 `sendAlimtalk(to, templateId, vars)` 정의해두면 추후 교체 쉬움.
- **템플릿 반려 리스크**: 에스테틱 특성상 "재방문 안내" 같은 문구가 광고성으로 판정 가능. 심사 1회 반려 가정하고 일정 짤 것.
- **발송 성공률**: SOLAPI는 웹훅으로 상태 업데이트. 반드시 상태 로그 테이블로 보존.

## 6. 초기 셋업 체크리스트 (SOLAPI 기준)

> 예상 총 소요: **영업일 기준 7~14일** (템플릿 심사 1회 반려 가정 시 +3일)

### Phase A. 사업자·채널 준비 (병렬 진행 가능, 2~5일)

- [ ] **사업자등록증** 준비 (개인사업자 포함. 미사업자는 알림톡 발송 **불가**)
- [ ] **대표자 신분증** 사본 준비 (비즈채널 인증용)
- [ ] **카카오톡 채널 개설** — kakao business 관리자 센터 (`business.kakao.com`)
  - 채널명: 트로이아르케 또는 브랜드명 (변경 제약 있음, 신중)
  - 프로필 이미지, 소개, 홈 URL
- [ ] **비즈니스 채널 인증 신청** — 사업자등록증 업로드, 심사 1~3일
- [ ] 인증 완료 후 **검색용 ID** 확인 (중계사 연동 시 필요)

### Phase B. SOLAPI 계정 세팅 (1일)

- [ ] [solapi.com](https://solapi.com) 회원가입 (이메일·휴대폰 인증)
- [ ] 가입 시 지급된 **300P 테스트 포인트** 확인
- [ ] **API Key / API Secret** 발급 (개발자 콘솔)
- [ ] `.env`에 `SOLAPI_API_KEY`, `SOLAPI_API_SECRET` 저장 (gitignore)
- [ ] 후불 결제 신청 (선택, 초기엔 충전으로 시작 가능)
- [ ] 사업자 정보 등록 (세금계산서용)

### Phase C. 카카오 채널 연동 (1일)

- [ ] SOLAPI 콘솔 → [카카오] → [채널 등록]
- [ ] 카카오 비즈 채널의 **검색용 ID 입력** → 연동
- [ ] 발급된 **PFID** 기록 (템플릿 등록·발송 시 사용)

### Phase D. 템플릿 심사 (2~5일, 반려 시 재신청)

- [ ] 템플릿 문구 초안 작성. 예시:
  - ```
    [트로이아르케] #{고객명}님, 다음 관리 예약일이 다가오고 있어요.
    일정: #{예약일시}
    케어: #{서비스명}
    예약 변경/취소는 답변 부탁드립니다.
    ```
- [ ] 변수는 `#{변수명}` 형태
- [ ] **광고성 표현 금지** ("할인", "이벤트", "특가" 등 → 알림톡 반려 사유)
- [ ] 카테고리 선택: "예약/서비스 이용" 관련
- [ ] SOLAPI 콘솔 또는 `createKakaoAlimtalkTemplate` API로 등록
- [ ] 카카오 심사 대기 (1~3 영업일)

### Phase E. Node.js 통합 (0.5~1일)

- [ ] `npm i solapi` (트로이아르케 CRM `package.json`에 추가)
- [ ] `src/integrations/kakao/alimtalkService.ts` 어댑터 파일 생성
  - 인터페이스: `sendAlimtalk(phone, templateId, variables)`
  - 내부 구현: SOLAPI SDK 호출
  - 실패 시 SMS로 폴백 (옵션)
- [ ] `.env`에 `SOLAPI_PFID`, `SOLAPI_TEMPLATE_ID_REMINDER` 추가
- [ ] 발송 로그 테이블 설계 (Supabase): `notification_logs(id, customer_id, channel, template_id, status, provider_msg_id, sent_at, error)`
- [ ] 웹훅 수신 엔드포인트 구현 (상태 업데이트 반영)

### Phase F. 검증 (0.5일)

- [ ] 테스트 번호로 알림톡 1건 발송 → 수신 확인
- [ ] 카카오톡 채널 차단 사용자 → 자동 SMS 폴백 동작 확인 (SOLAPI는 failover 옵션 지원)
- [ ] 수신 거부/차단 처리 프로세스 확인
- [ ] 리포트 대시보드에서 성공/실패 카운트 확인

### Phase G. 런칭 전 준수사항

- [ ] 개인정보처리방침에 "카카오 알림톡 발송을 위한 휴대폰 번호 이용" 명시
- [ ] 고객 약관에 알림톡 수신 동의 조항 (정보성은 옵트아웃 허용)
- [ ] 발송 시간대 제한 준수 (야간 제한은 광고성에만 해당, 정보성은 상대적으로 자유지만 운영 규칙 설정)
