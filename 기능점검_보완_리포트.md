## ✅ 오늘 보완 완료 (17건)

### 데이터 정합성 (돈·회차 숫자가 실제와 맞게) — 코어 수정
1. **결제 수정 시 고객 누적결제액 정합** — Sales에서 결제금액 고치면 고객 `totalSpent`(VIP 등급 기준)가 이전분 빼고 새 금액으로 재계산. (store.ts `PaymentStore.update`)
2. **시술기록 수정/삭제 시 프로그램 회차 정합** — 이전엔 시술기록을 고치거나 지워도 차감된 회차가 그대로거나 영구 손실됐음. 이제 수정 시 이전 연결 되돌리고 새로 차감, 삭제 시 회차 복구. (store.ts `TreatmentLogStore.update/delete`, `useSession` 클램프)

### 잘못된 저장 방지 (검증)
3. **결제 0원 falsy 버그** — 프로그램 등록에서 0원(무료 이벤트) 입력이 조용히 정가로 둔갑하던 것 수정. 잘못된 값은 차단. (Customers)
4. **신규 고객 전화번호 중복 방지** — 같은 번호 재등록 시 경고(기존 고객 안내). 이름·전화 필수 검증. (Customers)
5. **매출 음수/0원 결제 차단** — 마이너스 금액으로 매출·누적액 깎기 방지. (Sales)
6. **제품 음수 재고·가격 차단** — 재고/판매가/원가/최소재고 0 미만 저장 방지. (Products)
7. **직원 저장 무반응 → 필수값 안내** — 이름/전화 비우고 저장 시 "고장난 듯" 무반응이던 것, 안내 문구 표시(등록·수정 both). (Staff)

### 통계 정확도
8. **취소·노쇼가 통계에 섞이던 문제** — 대시보드 "오늘 예약" 건수, 직원별 "오늘 예약/이번달 매출"에서 취소·노쇼 제외(목록엔 그대로 표시). (Dashboard, Staff)
9. **시술기록 최신순 정렬** — 입력 순서라 최근 시술을 뒤져야 했음 → 최신순. (Treatments)

### 죽은 버튼·가짜 표시
10. **템플릿 "발송" 버튼 무반응** — 클릭해도 아무 일 없던 것, 발송 모달 열리게. (Messaging)
11. **사이드바 가짜 "네이버 예약 연동 중"** — 연동 안 됐는데 항상 "연동 중" 표시하던 것 → "정상 운영 중". (Sidebar)
12. **재고부족 배너 클릭 무반응** — 이제 클릭하면 "재고 부족만 보기" 필터 토글. (Products)

### 입력 오류·데이터 오염 방지
13. **온보딩 엑셀 업로드가 .xlsx를 깨서 읽던 문제** — 엑셀 바이너리를 텍스트로 잘못 파싱해 시술데이터 오염되던 것, 실제 파서(SheetJS)로 교체(xlsx/xls/csv 모두 지원). (Onboarding)
14. **마진율 미리보기 NaN/-Infinity** — 판매가 0 입력 시 미리보기가 깨지던 것 가드. (Products, Programs)

### 사고 예방
15. **예약 이중예약 방지** — 같은 담당자·같은 시간대에 겹치는 예약 등록 시 경고(강행 여부 확인). (Reservations)

*(코어(store.ts) 수정은 CORE_EDIT 절차 준수 + 호출자 전수 회귀점검 + build 검증 완료.)*

## ⏸️ 보류 — 원장님 결정/추가 정보 필요 (위험하거나 대공사)

### 🔴 반드시 확인 필요 (돈 관련)
- **실결제 PG가 테스트 모드 고정** (`payment.ts` `html5_inicis.INIpayTest`) — 구독 결제 시 실제 청구가 안 되는데도 "결제 성공"으로 처리됨. **실제 상용 결제를 받으려면 실 PG 계약·키가 필요.** 함부로 실 PG로 바꾸면 진짜 청구가 발생하므로 원장님 확인 후 진행. → NAS/상용화 단계에서 같이.
- **결제 후에도 "무료 체험 중" 배너 유지** — 플랜 결제 성공 시 `subscriptions`엔 기록되나 `branches.plan`이 안 바뀌어 앱은 계속 체험판으로 인식. 결제 로직·구독 연동은 상용화 결제 확정과 함께 손봐야 함(회귀 위험 커서 단독 수정 보류).

### 🟡 대공사 (별도 작업으로)
- **예약 "완료처리"가 시술기록/매출/회차를 자동 생성 안 함** — 지금은 완료 눌러도 별도로 시술기록을 또 입력해야 함. 자동화하려면 UX 설계 필요(중복입력 방지 포함). 우선 경고문구라도 넣을지 결정 필요.
- **메시지 수신대상 "직접 선택"이 실제로는 선택 UI 없음**, 세그먼트 카드(생일·미방문 등)가 전부 "전체" 대상으로 열림 — 발송 게이트웨이(NAS) 연동과 함께 정비 권장(오발송 방지).
- **상단 검색창·알림벨이 동작 안 함**(모든 화면 공통) — 전역 고객검색·알림 기능 신규 구현 필요.
- **관리자(Admin) 화면**: 사용자 role/계정 편집 기능 없음, Supabase 호출 에러처리 없음, 일부 상태카드 하드코딩("Connected" 등).

### 🟢 소소한 개선 (다음 차수)
- 등록 프로그램/시술기록을 고객 상세에서 바로 수정·삭제, 상세→고객 프로필 링크, 예약 30분 단위 슬롯, 예약 수정 시 구글캘린더 동기화, 피부유형/유입경로 직접입력, 전화번호 형식검증, 워크인 고객명 직접입력, 관리자 가격표 불일치(49,000 vs 29,000) 등.
