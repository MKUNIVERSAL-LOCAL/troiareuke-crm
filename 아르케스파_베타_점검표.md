# 아르케스파 CRM 베타 점검표

> 작성: 2026-07-14 · 대상: 아르케스파 베타 사용 (기존 클라우드 Supabase 기반 현행 릴리스)
> 이 문서는 현재 배포본(OneDrive exe) 코드 상태를 기준으로 작성됨. 코드 변경 없음.
> 베타 오류 정리 후 NAS 기능은 **관리자 계정 생성 API부터** 재개 예정.

## 1. 기능 구분 — 한눈에 보기

### ✅ 실제 사용 가능 (베타 검증 대상)

| 영역 | 기능 | 비고 |
|---|---|---|
| 고객 | 등록·수정·삭제, 등급/피부타입/태그, 검색, 엑셀 일괄 업로드, 상담 기록 | Supabase 동기화 — 여러 PC에서 공유됨 |
| 예약 | 등록·수정·취소, 상태 변경(대기/확정/완료/노쇼), 예약경로 표기(네이버·카카오 등은 **수동 선택**) | 〃 |
| 시술 | 시술기록 등록·수정·삭제, 회차 관리, 다음 방문 권장일 | 〃 |
| 시술 사진 | 전/후 사진 업로드·타임랩스 보기 | ⚠️ **그 PC에만 저장됨** — 아래 3-경고 참조 |
| 재고 | 제품 등록·수정·삭제, 재고/최소재고, 부족 알림, 판매 시 재고 차감 | Supabase 동기화 |
| 매출 | 결제(수납) 기록, 할인 입력, 결제수단별 기록, 매출 통계·대시보드 | 취소·노쇼는 통계에서 제외되는지가 점검 포인트 |
| 권한 | superadmin / admin / staff 3단계. **staff는 고객 전화번호 마스킹**(010-\*\*\*\*-1234) | 화면 마스킹 — 베타 검증 대상 |
| 재방문 리마인더 | 대시보드에서 재방문 지난 고객 자동 산출 + 메시지 초안 **복사** | 자동 발송 아님 — 복사해서 직접 카톡/문자 |
| 기타 | 로컬 백업(설정>백업), 온보딩, 홈케어 가이드 | |

### ⛔ 사용 금지 (베타 중 절대 누르지/쓰지 말 것)

| 기능 | 위치 | 이유 |
|---|---|---|
| 🚨 **구독/플랜 결제** | 설정 > 구독 (플랜 변경·결제 버튼) | 포트원 **테스트 PG**에 고정되어 실청구는 되지 않지만, 결제 뒤 지점 플랜 상태가 완전히 동기화되지 않음. 베타 데이터 오염 방지를 위해 사용 금지 |
| 메시지 발송 (SMS/LMS/카카오) | 메시지 페이지 "발송", 설정 SMS 테스트, 고객 상담 "카카오 전송" | 게이트웨이 미연동 — 실제 발송 안 되고 이력에 '실패'로만 기록됨. 고객에게 안 감 |
| 예약 발송 | 메시지 페이지 | 미구현 (안내 알림만 뜸) |
| 지점 관리자 계정 생성 | Admin > 지점 관리 | API 미연동 — 지점은 생성되지만 계정은 '대기'로 남음 |
| 네이버 예약 자동 연동 | 설정 > 연동 | 미구현 ("준비 중" 표기). 네이버 예약은 **수동 입력**으로 |

## 2. 베타 시작 전 준비 (관리자 1회)

- [ ] 사용 PC 확정 — **시술 사진은 업로드한 PC에서만 보이므로**, 사진 쓸 PC는 1대로 고정 권장
- [ ] 계정 준비: 원장용 admin 1개, 직원용 staff 계정(직원 수만큼)
- [ ] 로그인 확인: 각 계정으로 로그인 → 대시보드 표시 확인
- [ ] 설정 > 매장 정보(상호·연락처) 입력
- [ ] 샘플 데이터와 실데이터 구분 인지 (샘플은 통계·리마인더에서 자동 제외됨)
- [ ] **설정 > 백업 > 지금 백업**을 1회 실행하고 파일 생성 확인 (이후 매일 마감 시 1회)

## 3. 직원용 테스트 순서 — 실제 업무 흐름 그대로

각 단계에서 "예상 결과"와 다르면 **§5 오류 기록 양식**에 적고 다음 단계로 계속 진행 (멈추지 않음).

### 시나리오 A. 신규 고객 첫 방문 (핵심 흐름 — 매일 반복)

| # | 하는 일 | 예상 결과 | 확인 |
|---|---|---|---|
| A1 | 고객 > 신규 등록 (이름·전화·피부타입·유입경로) | 목록에 즉시 표시 | ☐ |
| A2 | 예약 > 신규 예약 (A1 고객, 오늘 날짜, 시술·담당자 선택) | 예약 목록/캘린더에 표시, 시간 겹침 시 안내 | ☐ |
| A3 | 고객 방문 → 예약 상태를 '확정'→'완료'로 변경 | 상태 뱃지 변경 | ☐ |
| A4 | 시술 > 시술기록 등록 (고객·시술·회차·메모, **전 사진 업로드**) | 기록 저장, 사진 표시, 고객 상세에 방문 이력 반영 | ☐ |
| A5 | 매출 > 결제 등록 (금액·할인·결제수단) | 결제 기록 생성, **고객 누적 결제액 증가**, 대시보드 오늘 매출 반영 | ☐ |
| A6 | 고객 상세에서 방문수·누적액 확인 | A4·A5와 숫자 일치 | ☐ |

### 시나리오 B. 재방문 고객 + 제품 판매

| # | 하는 일 | 예상 결과 | 확인 |
|---|---|---|---|
| B1 | 기존 고객 검색 (이름 일부/전화 뒷자리) | 검색 결과 정확 | ☐ |
| B2 | 시술기록 등록 + **후 사진 업로드** → 타임랩스 보기 | 전/후 사진이 날짜순으로 재생 | ☐ |
| B3 | 제품 > 재고 확인 후, 결제 시 제품 판매 포함 | **재고가 판매 수량만큼 차감**, 재고보다 많이 팔면 차단 | ☐ |
| B4 | 재고를 최소재고 이하로 만들기 | 재고 부족 알림 표시 | ☐ |

### 시나리오 C. 수정·취소·노쇼 (정합성 — 오류 잘 나오는 곳)

| # | 하는 일 | 예상 결과 | 확인 |
|---|---|---|---|
| C1 | 결제 금액 수정 | **고객 누적액이 차액만큼 정확히 조정** (이중 가산 없음) | ☐ |
| C2 | 결제 삭제 | 누적액에서 정확히 차감, 매출 통계에서 제거 | ☐ |
| C3 | 예약 '취소' 처리 | **매출·방문 통계에 포함 안 됨**, 재방문 리마인더에서 미래예약으로 취급 안 됨 | ☐ |
| C4 | 예약 '노쇼' 처리 | 통계에서 제외 | ☐ |
| C5 | 시술기록 수정(회차 변경)·삭제 | 회차가 꼬이지 않음 | ☐ |
| C6 | 고객 삭제 | 관련 화면에서 사라짐, 대시보드 오류 없음 | ☐ |

### 시나리오 D. 권한 분리 (admin PC와 staff 계정으로)

| # | 하는 일 | 예상 결과 | 확인 |
|---|---|---|---|
| D1 | **staff 계정** 로그인 → 고객 목록·상세·예약·시술·리마인더 카드 | 전화번호가 전부 `010-****-1234` 형태로 마스킹 | ☐ |
| D2 | staff 계정에서 화면 곳곳에서 마스킹 안 된 번호 찾아보기 | **한 곳이라도 원본 노출되면 즉시 기록** (개인정보 이슈 — 우선순위 높음) | ☐ |
| D3 | staff 계정에서 Admin 메뉴 접근 시도 | 접근 불가 | ☐ |
| D4 | **admin 계정** 로그인 | 전화번호 원본 표시, 전체 메뉴 접근 가능 | ☐ |

### 시나리오 E. 멀티 PC·재시작 (동기화)

| # | 하는 일 | 예상 결과 | 확인 |
|---|---|---|---|
| E1 | PC①에서 고객 등록 → PC②(또는 앱 재시작) 후 확인 | 고객·예약·시술·결제·재고 모두 보임 | ☐ |
| E2 | PC①에서 올린 시술 사진을 PC②에서 확인 | **안 보이는 게 현재 정상** (기기별 저장) — 보이면 오히려 기록 | ☐ |
| E3 | 인터넷 끊고 앱 사용 → 다시 연결 | 오프라인 배너 표시, 재연결 후 데이터 정상 | ☐ |

### 시나리오 F. 하루 마감 루틴 (매일)

| # | 하는 일 | 예상 결과 | 확인 |
|---|---|---|---|
| F1 | 대시보드에서 오늘 매출·방문·예약 수를 수기 계산과 대조 | 일치 | ☐ |
| F2 | 재방문 리마인더 카드 확인 → 대상 고객에게 초안 복사 → **개인 카톡/문자로 직접 발송** | 초안 복사 정상 (CRM 발송 버튼은 사용 금지) | ☐ |
| F3 | 설정 > 백업 > 지금 백업 | 백업 파일 생성 | ☐ |

## 4. 베타 중 행동 수칙 (직원 공지용)

1. 🚨 **설정 > 구독의 결제·플랜변경 버튼은 누르지 않는다** (테스트 PG이며 구독 상태 동기화가 미완성).
2. 메시지 "발송" 버튼은 눌러도 고객에게 안 간다 — 고객 안내는 **초안 복사 → 개인 폰으로 직접** 발송.
3. 시술 사진은 **정해진 PC 1대**에서만 올리고 본다.
4. 오류가 나도 앱을 지우거나 데이터를 임의 삭제하지 말고 **§5 양식에 기록 후 계속 사용**.
5. 매일 마감 때 백업 1회.
6. 실고객 개인정보가 들어가므로, staff 화면에서 전화번호 원본이 보이면 **그날 바로 보고**.

## 5. 오류 기록 양식

오류 1건 = 1행. 파일은 `아르케스파_베타_오류로그.md`(또는 인쇄물/시트)에 누적.

| 항목 | 기록할 내용 |
|---|---|
| 번호 | B-001부터 순번 |
| 날짜/시각 | 예: 07-15 14:30 |
| 작성자·계정 | 이름 + admin/staff 여부 |
| 화면 | 예: 고객 상세, 예약 캘린더, 매출 통계 |
| 하려던 일 | 한 줄. 예: "결제 금액 5만→3만으로 수정" |
| 실제 일어난 일 | 그대로. 오류 메시지는 **토씨까지 그대로**, 가능하면 스크린샷 |
| 기대한 결과 | 예: "누적액이 2만 줄어야 함" |
| 재현 여부 | 같은 조작 다시 했을 때 또 나는지 (예/아니오/가끔) |
| 심각도 | 🔴 업무 불가·데이터 오염·개인정보 노출 / 🟡 불편하지만 우회 가능 / 🟢 사소(오타·표시) |
| 데이터 단서 | 관련 고객명·예약날짜 등 (재현용) |

### 작성 예시

> **B-001** · 07-15 14:30 · 김직원(staff) · 화면: 고객 상세
> 하려던 일: 결제 5만 원 삭제
> 실제: 삭제됐는데 누적 결제액이 그대로임
> 기대: 누적액 5만 감소
> 재현: 예 (다른 고객도 동일)
> 심각도: 🔴 · 단서: 고객 '홍길동', 7/15 결제건

### 심각도별 처리

- 🔴 발견 즉시 원장 보고 → 당일 개발자(오너) 전달
- 🟡/🟢 오류로그에 누적 → 주 1회 모아서 전달

## 6. 베타 종료 판정 기준 (참고)

- 시나리오 A~F를 **서로 다른 날 3회 이상** 완주, 🔴 미해결 0건
- 대시보드 매출·방문 수치가 수기 대조와 3일 연속 일치
- staff 마스킹 누락 0건
- 이후: 오류 정리 → 수정 → **NAS 관리자 계정 생성 API** 구현 재개
