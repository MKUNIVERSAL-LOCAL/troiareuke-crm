# 상용화 검토 기록 — 2026-07-15

기준: PR #6 (`feat/nas-central-backend`) 전체와 PR #8 (`판매 고객 비밀번호 찾기 복구`) 비교.

## P0

- **NAS API가 CSP에서 차단됨**: `VITE_AUTH_API_URL` origin을 빌드 CSP에 검증·주입하도록 수정했다.
- **기존 판매 고객 컷오버 미검증**: Supabase 계정·데이터는 NAS로 자동 이관되지 않는다. 합성 데이터
  리허설과 백업 승인 전에는 `VITE_NAS_CUTOVER_APPROVED=true` 없이는 NAS 연결 빌드가 실패한다.
- **NAS 관리자 지점 발급이 Supabase 세션에 의존**: 지점 조회·생성·수정·비활성화를 NAS 계정 API로
  전환했다. 비활성화는 고객 데이터를 삭제하지 않고 소속 계정 세션만 종료한다.

## P1

- **오프라인 쓰기 유실 가능**: 로컬 변경을 지점별 outbox에 먼저 기록하고 원격 로드·로그아웃 전에
  순서대로 재전송한다. 사진도 전송 전에 dirty로 표시하고 로그아웃 전에 완료를 확인한다. 실패 시
  원격 데이터를 읽거나 로컬 캐시를 지우지 않아 오래된 서버 값이 로컬을 덮지 않는다.
- **최초 이관 부분 성공 오인**: 500행 청크 fire-and-forget을 제거하고 최대 2,000행 단일 트랜잭션
  요청을 기다린 뒤 완료로 처리한다. 초과 데이터는 관리자 이관을 요구하며 로컬을 유지한다.
- **프록시 신뢰/rate-limit 우회 가능성**: API 포트를 loopback에만 바인딩하고 프록시 hop을 명시한다.
- **SMTP 장애가 전체 API 기동 차단**: SMTP 확인 실패는 경고로 남기되 로그인·데이터 API는 기동한다.
- **복구 링크의 하위 경로·Electron·GitHub Pages 실패**: PR #8의 URL 검증과 SPA 경로 복원을 선별 통합했다.
