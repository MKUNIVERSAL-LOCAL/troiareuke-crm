# 트로이아르케 CRM — 빌드·배포 런북

> **목적**: 우리가 코드에 반영한 작업이 **사용자가 실제 실행하는 데스크톱 CRM(OneDrive 실행본)에 전부 적용**되도록, 매 작업 종료 시 따르는 표준 절차. 이 절차를 건너뛰면 코드는 바뀌었는데 실행 화면은 그대로인 문제가 생긴다.

---

## 0. 소스·실행본 위치 (혼동 금지)

| 구분 | 경로 | 용도 |
|------|------|------|
| **편집 정본 소스** | `C:\dev\troiareuke-crm` (git clone) | 모든 코드 수정은 여기서 |
| GitHub 원격 | `github.com/MKUNIVERSAL-LOCAL/troiareuke-crm` | 정본, PR 경유 머지 |
| **실행본(배포 대상)** | `F:\OneDrive\바탕 화면\06_개발프로젝트\전체 작업폴더\트로이아르케-CRM` | 사용자가 `트로이아르케 CRM.exe`로 실행 |

⚠️ OneDrive 폴더의 `src`는 없다 — **빌드 산출물만** 있음. 편집은 반드시 `C:\dev`에서.

---

## 1. 코드 자가 검증 (원칙 3) — 반드시 먼저

```bash
cd /c/dev/troiareuke-crm
npx tsc --noEmit        # TS 에러 0 확인
npm run build           # vite 프로덕션 빌드 성공 확인
```

두 개 다 통과해야 다음 단계로. 하나라도 실패 시 배포 금지.

---

## 2. 데스크톱 실행본 재빌드

```bash
cd /c/dev/troiareuke-crm
npm run electron:build:dir
```

- 산출물: `C:\dev\troiareuke-crm\release\win-unpacked\`
- 핵심 파일: `트로이아르케 CRM.exe`, `resources\app.asar` (← 우리 코드가 담기는 곳)

---

## 3. OneDrive 실행본으로 반영 (robocopy)

⚠️ **절대 `/MIR`(미러) 쓰지 말 것.** OneDrive 폴더에는 `아르케스파 고객 데이터`, `crm솔루션 이미지` 등 **win-unpacked에 없는 폴더**가 함께 있어, 미러하면 고객 데이터가 삭제된다. 반드시 **덮어쓰기(/E, 삭제 없음)** 만 사용.

PowerShell에서:

```powershell
$src = "C:\dev\troiareuke-crm\release\win-unpacked"
$dst = "F:\OneDrive\바탕 화면\06_개발프로젝트\전체 작업폴더\트로이아르케-CRM"
# 앱 코드(app.asar)와 실행 파일·런타임 일체를 덮어쓰기(삭제 없이)
robocopy $src $dst /E /XO:$false /NFL /NDL /NP
```

- 최소 반영만 원하면 코드가 담긴 `resources\app.asar` 1개 + `트로이아르케 CRM.exe` 만 복사해도 된다.
- **실행 중이면 exe/app.asar가 잠겨 복사 실패** → CRM 창을 완전히 닫고 재시도.

---

## 4. 반영 확인

1. OneDrive 폴더의 `트로이아르케 CRM.exe`, `resources\app.asar` **수정시각이 방금**인지 확인.
2. `트로이아르케 CRM.exe` 실행 → 변경한 화면이 실제로 바뀌었는지 눈으로 확인.
